# دليل استقرار الاتصال - Connection Stability Guide

## نظرة عامة
هذا الدليل يشرح كيفية ضمان استمرار اتصال التطبيق مع الخادم `wss://nopoh22-sms-relay-server.hf.space/device` بدون انقطاع.

---

## 🔧 الإصلاحات المطبقة

### 1. تحسين RelayClient
✅ **إعادة الاتصال التلقائي المحسّن**
- زيادة عدد محاولات إعادة الاتصال إلى 50 محاولة
- تأخير متزايد مع عشوائية لتجنب الضغط على الخادم
- إعادة تعيين العداد عند توفر شبكة جديدة

✅ **Heartbeat محسّن**
- إرسال ping كل 20 ثانية بدلاً من 30
- مراقبة فشل إرسال الـ heartbeat
- إعادة الاتصال التلقائي عند فشل الـ heartbeat

✅ **إدارة أفضل للطاقة**
- WakeLock لمنع النوم أثناء الاتصال
- تحرير WakeLock عند قطع الاتصال
- مراقبة حالة البطارية

✅ **مراقبة الشبكة**
- تسجيل مراقب للشبكة (WiFi + Cellular)
- إعادة الاتصال التلقائي عند عودة الشبكة
- مراقبة تغييرات قدرات الشبكة

### 2. ConnectionMonitor الجديد
✅ **مراقبة مستمرة**
- فحص حالة الاتصال كل 30 ثانية
- إعادة اتصال قسرية كل 5 دقائق إذا لزم الأمر
- مراقبة حالة الخدمة وإعادة تشغيلها

### 3. تحسين PaymentGatewayService
✅ **خدمة مقاومة للقتل**
- START_STICKY لإعادة التشغيل التلقائي
- Foreground Service مع إشعار دائم
- مراقبة حالة جميع المكونات

## 📱 إعدادات الهاتف المطلوبة

### 1. إيقاف Battery Optimization
```
الإعدادات → البطارية → تحسين البطارية → SMS Payment Gateway → عدم التحسين
```

### 2. السماح بالعمل في الخلفية
```
الإعدادات → التطبيقات → SMS Payment Gateway → البطارية → السماح بالنشاط في الخلفية
```

### 3. Auto-Start (حسب الشركة المصنعة)

#### Xiaomi/MIUI:
```
الإعدادات → التطبيقات → إدارة التطبيقات → SMS Payment Gateway → التشغيل التلقائي → تفعيل
الإعدادات → التطبيقات → SMS Payment Gateway → أذونات أخرى → البدء في الخلفية → السماح
```

#### Huawei/EMUI:
```
الإعدادات → التطبيقات → SMS Payment Gateway → البطارية → App Launch → إدارة يدوية
تفعيل: Auto-launch, Secondary launch, Run in background
```

#### Samsung:
```
الإعدادات → العناية بالجهاز → البطارية → تحسين الاستخدام → SMS Payment Gateway → عدم التحسين
الإعدادات → التطبيقات → SMS Payment Gateway → البطارية → السماح بالنشاط في الخلفية
```

#### OnePlus/OxygenOS:
```
الإعدادات → البطارية → تحسين البطارية → SMS Payment Gateway → عدم التحسين
الإعدادات → التطبيقات → SMS Payment Gateway → البطارية → السماح بالنشاط في الخلفية
```

### 4. إعدادات الشبكة
```
الإعدادات → WiFi → متقدم → Keep WiFi on during sleep → Always
الإعدادات → Mobile Networks → Always keep mobile data on
```

## 🔍 مراقبة حالة الاتصال

### في التطبيق:
1. افتح التطبيق
2. اذهب إلى الإعدادات
3. تحقق من "Connection Status"
4. يجب أن يظهر "Connected" باللون الأخضر

### في Logs:
```bash
# عرض logs الاتصال
adb logcat | grep "RelayClient\|ConnectionMonitor"

# البحث عن رسائل معينة
adb logcat | grep "WebSocket متصل بنجاح"
adb logcat | grep "Heartbeat sent"
```

## 🚨 استكشاف الأخطاء

### المشكلة: الاتصال ينقطع كل فترة
**الحلول:**
1. تحقق من إعدادات Battery Optimization
2. تأكد من أن WiFi لا ينام
3. فعّل Auto-Start للتطبيق
4. أعد تشغيل الخدمة من التطبيق

### المشكلة: لا يعيد الاتصال تلقائياً
**الحلول:**
1. تحقق من أذونات الشبكة
2. تأكد من أن الخدمة تعمل (إشعار دائم)
3. أعد تشغيل التطبيق
4. تحقق من logs للأخطاء

### المشكلة: الاتصال يفشل نهائياً
**الحلول:**
1. تحقق من اتصال الإنترنت
2. تحقق من صحة API Key
3. تحقق من حالة الخادم
4. امسح cache التطبيق وأعد المحاولة

## 📊 مؤشرات الاتصال الصحي

### ✅ اتصال صحي:
- `✅ WebSocket متصل بنجاح` في logs
- `💓 Heartbeat sent` كل 20 ثانية
- `💓 Pong received - الاتصال سليم` كاستجابة
- إشعار الخدمة يظهر في شريط الإشعارات

### ⚠️ اتصال غير مستقر:
- `⚠️ فشل إرسال Heartbeat` في logs
- `⏰ سيتم إعادة الاتصال بعد X ثانية` متكرر
- انقطاع الاتصال كل بضع دقائق

### ❌ اتصال فاشل:
- `❌ فشل الاتصال` في logs
- `🚫 وصلت محاولات إعادة الاتصال إلى الحد الأقصى`
- عدم وجود إشعار الخدمة

---

## 🔄 إعادة تشغيل الخدمة

### من التطبيق:
1. افتح التطبيق
2. اضغط "Stop Service"
3. انتظر 5 ثوانِ
4. اضغط "Start Service"

### من ADB:
```bash
# إيقاف الخدمة
adb shell am force-stop com.sms.paymentgateway

# تشغيل التطبيق
adb shell am start -n com.sms.paymentgateway/.presentation.ui.MainActivity
```

### إعادة تشغيل الهاتف:
- الخدمة ستبدأ تلقائياً بعد إعادة التشغيل (BootReceiver)

## 📈 تحسينات الأداء

- Heartbeat كل 20 ثانية بدلاً من 10
- WakeLock جزئي فقط عند الحاجة
- إيقاف المراقبة عند عدم الحاجة

### 2. تحسين الشبكة:
- إعادة استخدام الاتصال
- ضغط الرسائل
- تجميع الطلبات

### 3. إدارة الذاكرة:
- تنظيف logs القديمة
- إدارة cache الرسائل
- تحرير الموارد غير المستخدمة

## 🛡️ الأمان

- مصادقة بـ API Key

- Foreground Service

- ConnectionMonitor

   - كم مرة
   - الخطوات المتخذة

## 🎯 الخلاصة

مع هذه الإصلاحات، يجب أن يبقى الاتصال مستقراً لفترات طويلة. المفاتيح الرئيسية:

1. **إعدادات الهاتف صحيحة** (Battery Optimization + Auto-Start)
2. **الخدمة تعمل كـ Foreground Service**
3. **مراقبة مستمرة وإعادة اتصال تلقائي**
4. **إدارة ذكية للطاقة والشبكة**

إذا اتبعت هذه التعليمات، لن تحتاج لفتح الهاتف وإعادة الاتصال يدوياً مرة أخرى! 🚀
